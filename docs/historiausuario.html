Historias de usuario

Acá una breve descripción de los Requerimientos Funcionales de ProjectParking

Historia de usuario 1

Descripción: Como Usuario quiero que el sistema me permita registrarme correctamente, para que se puede consultar parqueaderos disponibles.

Usuarios: Arrendatario o Conductor.

Puntos estimados: 13.

Criterios de aceptación: Los datos del usuario se mantendrán bajo una seguridad, en el cual le pedirá el correo electrónico y la contraseña.

Historia de usuario 2

Descripción: Como usuario quiero que el sistema me envié un correo electrónico para el arrendatario y conductor, notificando la reserva de un estacionamiento con los siguientes datos: Ubicación, Hora llegada, hora fin, nombre arrendatario, valor pagado o por pagar y placas de vehículo para saber los datos de la reserva y que sea más oficial.

Usuarios: Arrendatario o Conductor.

Puntos estimados: 8.

Criterios de aceptación: El usuario recibirá el correo electrónico en máximo un minuto en el cual es una forma más eficiente de oficializar un estacionamiento con los datos correspondientes.

Historia de usuario 3

Descripción: Como usuario quiero que el sistema permita seleccionar cualquiera de los siguientes métodos de pago: tarjetas de crédito/debito, PayPal, efectivo y generar el pago de la reserva para así poder concluir con su reserva.

Usuarios: Arrendatario o Conductor.

Puntos estimados: 13.

Criterios de aceptación: El usuario podrá escoger el método de pago preferido.

Historia de usuario 4

Descripción: Como usuario quiero que cuando se realice una reserva de estacionamiento el sistema deberá mostrar el estacionamiento en modo reservado durante el tiempo reservado y la disponibilidad próxima del estacionamiento para que no haya una operación duplicada y así dos usuarios no acepten el mismo estacionamiento.

Usuarios: Arrendatario o Conductor.

Puntos estimados: 8.

Criterios de aceptación: Dos usuarios no podrán tener el mismo estacionamiento en el tiempo que se hizo la reserva anterior.

Historia de usuario 5

Descripción: Como usuario quiero que El sistema deberá mostrar la ubicación de los estacionamientos disponibles dentro de un rango seleccionado a través de un mapa para que se facilite la comodidad del usuario.

Usuarios: Arrendatario o Conductor.

Puntos estimados: 5.

Criterios de aceptación: Para beneficio del usuario se facilita los estacionamientos en el mapa para ahorrar tiempo y costo.

Historia de usuario 6

Descripción: Como Usuario quiero que el el sistema permitirá generar una calificación del arrendatario al conductor y del conductor al arrendatario basado en los siguientes criterios: ubicación, espacio, precio, seguridad, comodidad para que pueda tener una opinión por medio de esta calificación el uno al otro.

Usuarios: Arrendatario o Conductor.

Puntos estimados: 2.

Criterios de aceptación: El usuario tendrá una perspectiva de arrendatario a conductor o conductor a arrendatario y esta calificación se podrá ver en diferentes usuarios.
